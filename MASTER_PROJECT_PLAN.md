## 🗄️ DATABASE ARCHITECTURE

### Core Tables
```sql
-- User Management
profiles (id, email, subscription_status, plan_type, subscription_id)

-- Content Management
blog_posts (id, title, slug, content, category, tags, engagement_score)
migration_requests (id, name, email, company, current_tool, status)

-- Analytics
analytics_events (id, session_id, user_id, event, properties, url)
share_views (id, session_id, page, meta, user_agent)

-- Performance (implied from code)
creatives (id, brand_id, assets, performance_prediction, fraud_analysis)
campaigns (id, brand_id, platform, status)
performance_data (id, creative_id, campaign_id, metrics)
touchpoints (id, user_id, creative_id, campaign_id, touchpoint_type)
brands (id, name, brand_model)
```

### Key Features
- ✅ Row Level Security (RLS) enabled on all tables
- ✅ Performance indexes on commonly queried columns
- ✅ Full-text search with pg_trgm extension
- ✅ JSONB columns for flexible data storage
- ✅ Automated timestamps and UUID generation
