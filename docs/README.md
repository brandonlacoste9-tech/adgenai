## 📝 Contributing to Documentation

When adding new documentation:

1. **Place files in appropriate category**:
   - Strategy docs: Use descriptive names with `-strategy` or `-analysis` suffix
   - Technical docs: Use descriptive names with `-architecture` or `-guide` suffix
   - Process docs: Use descriptive names with `-process` or `-workflow` suffix

2. **Update this README**: Add your new document to the index above

3. **Use consistent formatting**:
   - Start with a clear title and overview
   - Use headers for organization
   - Include examples where helpful
   - Add links to related documents

4. **Keep it current**: Update docs when processes or strategies change
