# Usage Worker Setup

## File Roles

1. **`usage.py`**:
   - API route to enqueue usage events.
   - Connects to Redis queue.

2. **`usage_worker.py`**:
   - Worker that processes usage events.
   - Submits usage records to Stripe.

---

## Redis Cloud Setup

1. Use **Upstash** (free tier, serverless):
   - Go to: [https://upstash.com/redis](https://upstash.com/redis)
   - Create a Redis database (region close to Render).
   - Copy your **Redis URL** (e.g., `rediss://:password@hostname:port`).

2. Set the Redis URL in Render as an environment variable:
   - `REDIS_URL = rediss://...`

3. Update your code:
   - Replace Redis connection in `usage.py` and `usage_worker.py`:

     ```python
     from redis import Redis

     redis_conn = Redis.from_url(os.getenv("REDIS_URL"))
     ```

---

## Worker Startup

1. Install dependencies:

   ```bash
   pip install fastapi redis rq stripe
   ```

2. Start the worker:

   ```bash
   rq worker usage --path ./backend
   ```

## Test Script

Use the following script to test the `/usage/enqueue` endpoint:

```python
import requests

API_URL = "http://localhost:8000/usage/enqueue"

payload = {
    "user_id": "user_123",
    "gb_used": 0.75,
    "subscription_item_id": "si_test_ABC123"
}

response = requests.post(API_URL, json=payload)
print("Status:", response.status_code)
print(response.json())
```

## Stripe Usage Record Logic

1. **Endpoint**: `/v1/subscription_items/{id}/usage_records`
2. **Action**: Increment usage for a subscription item.
3. **Example**:

   ```python
   stripe.UsageRecord.create(
       subscription_item=subscription_item_id,
       quantity=int(gb_used * 1000),  # Convert GB to MB if needed.
       timestamp=int(time.time()),
       action="increment"
   )
   ```
